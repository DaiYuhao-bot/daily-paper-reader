## 摘要
在生成帧之间保持3D空间一致性的视频世界模型通常依赖于在RGB空间中构建的显式点云记忆。这种设计既计算昂贵（需要重复渲染和VAE编码），又本质上有损，因为通过像素空间的往返过程会丢弃学习到的潜在表示中的丰富特征。本文引入了用于视频世界模型的潜在空间记忆，这是一种持久化的3D缓存，直接在扩散潜在空间中存储场景信息，避免了像素空间重建。在此基础上，我们提出了Mirage，一个潜在空间空间记忆框架，通过深度引导反投影将潜在标记提升到3D中来构建记忆，并通过直接潜在空间扭曲合成新视图来查询记忆。这种统一公式消除了像素空间重建的信息损失以及重复编码和渲染的计算负担。实验表明，相对于显式3D基线，潜在空间记忆实现了高达10.57倍的端到端视频生成加速和55倍的内存占用减少。利用扩散模型的几何先验，Mirage在WorldScore上达到了最先进的性能，并在RealEstate10K上达到了强大的重建质量。

## Abstract
Video world models that maintain 3D spatial consistency across generated frames typically rely on explicit point cloud memory constructed in RGB space. This design is both computationally expensive, requiring repeated rendering and VAE encoding, and inherently lossy, as the round trip through pixel space discards rich features of the learned latent representation. In this paper, we introduce \emph{latent spatial memory} for video world models, a persistent 3D cache that stores scene information directly in the diffusion latent space, avoiding pixel-space reconstruction. Building on this, we propose Mirage, a latent-space spatial memory framework that constructs the memory by lifting latent tokens into 3D via depth-guided back-projection and queries it by synthesizing novel views through direct latent-space warping. This unified formulation eliminates both the information loss of pixel-space reconstruction and the computational burden of repeated encoding and rendering. Experiments show that latent spatial memory achieves up to \textbf{10.57}$\times$ faster end-to-end video generation and \textbf{55}$\times$ reduction in memory footprint relative to explicit 3D baselines. Leveraging the geometric prior of the diffusion model, Mirage attains state-of-the-art performance on WorldScore and strong reconstruction quality on RealEstate10K.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **背景**：大规模视频扩散模型已能生成逼真序列，但在长时生成或大幅相机运动下，缺乏显式的3D空间记忆会导致几何漂移（geometric drift），即帧间空间不一致。
- **现有方案的瓶颈**：当前视频世界模型通常采用**RGB空间中的显式点云记忆**，通过每步渲染成图像再经VAE编码回潜空间注入扩散模型。这带来两大问题：
  - **计算昂贵**：每步需高分辨率光栅化+VAE编码，随着记忆积累，成本急剧上升。
  - **信息有损**：像素空间往返会丢弃潜空间中丰富的特征（如语义、纹理），引入渲染伪影、VAE重建误差和分布不匹配。
- **核心问题**：能否在扩散模型的**原生潜空间**中直接构建和查询3D记忆，既消除像素往返的计算开销，又保留特征的完整性？

### 2. 论文提出的方法论：核心思想、关键技术细节、流程

- **核心思想**：提出**潜空间空间记忆（Latent Spatial Memory）**，将场景信息以“世界坐标+潜特征”的形式存储在3D缓存中，所有读/写操作均在VAE潜空间完成，避免像素空间转换。
- **关键技术细节**：
  - **记忆表示**：`M = {(pᵢ, fᵢ)}`，其中 `pᵢ ∈ R³` 是世界坐标，`fᵢ ∈ R^C` 是VAE编码器输出的潜特征（C=48），与扩散骨干的输入空间一致。
  - **记忆构建（写）**：对观察帧先VAE编码得到潜张量 `z`，利用单目深度估计网络（如Depth Anything 3）获得深度图，将潜网格每个单元通过**深度引导反投影**（公式4）提升到3D：
    ```
    p_uv = π⁻¹(u, v, D(u,v); K, E)   // 针孔反投影
    F_uv = z[:, v, u]
    ```
    每个潜单元产生一个记忆点。初始帧构建初始缓存，后续每块生成后更新。
  - **记忆读取（读）**：对目标视角 `(Eₜ, Kₜ)`，将所有记忆点投影到目标潜网格，经Z-buffer保留最前点，直接检索其潜特征（公式5）：
    ```
    iₜ(u,v) = argmin_i∈Ωₜ(u,v) [Eₜ pᵢ]_z
    ẑₜ(u,v) = F_{iₜ(u,v)}
    ```

    同时输出二进制可见性掩码 `mₜ`，指示哪些潜单元有投影点。
  - **注入方式**：读取的 `ẑₜ` 和 `mₜ` 通过**ControlNet风格侧分支**注入主干，无需额外编码器。

### 7. 优点

- **创新性**：首次提出在潜空间中存储和查询3D空间记忆，完全避免了像素空间往返，从根本上解决了计算和表示瓶颈。
- **有效性**：在多个基准上达到SOTA或竞争性能，同时大幅降低计算和内存开销。
- **实用性**：基于现有扩散模型（Wan2.2）进行微调，适配策略轻量（仅侧分支+LoRA），可推广到其他潜空间扩散模型。
- **可解释性**：可视化潜特征的主成分分析显示，存储的特征形成了语义聚类（墙壁、家具等），表明潜特征比RGB携带更丰富的信息。

### 8. 不足与局限

- **动态物体处理粗糙**：目前仅通过分割排除动态物体（人、车、天空），无法维持这些动态实体的长期状态，在运动频繁的场景中记忆帮助有限。
- **依赖深度估计**：构建缓存依赖于单目深度（Depth Anything 3），深度误差会传播到记忆，虽然在实验中鲁棒，但极端场景下可能失效。
- **训练数据单一**：主要基于RealEstate10K训练，该数据集多为静态场景，对复杂动态场景的泛化能力未经充分验证。
- **未与其他骨干验证**：仅测试了Wan2.2一种骨干，泛化到其他扩散模型（如Stable Video Diffusion、CogVideo）未知。
- **长时记忆管理**：缓存随生成无限增长，虽显存占用低但缺乏显式合并或剪枝策略，超长序列可能仍会累积冗余点。
