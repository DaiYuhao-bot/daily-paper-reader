## 摘要
近期将视觉-语言模型（VLM）作为生成模型条件提示编码器的方法，通常依赖于昂贵的端到端训练，或将特征映射到压缩表示，从而丢弃了诸如3D资产生成等几何感知任务所需的密集空间结构。为解决此问题，我们提出GAP3D，一种基于扩散的模块化方法，该方法直接将VLM生成的潜在表示与预训练图像编码器的完整补丁级特征空间对齐，从而使冻结的下游生成模型能够利用VLM作为提示编码器，同时保持空间结构化的条件信号。在3D资产生成评估中，我们的方法主要通过训练通用域图像-文本对，绕过了对大规模3D数据的需求。尽管仅针对文本输入进行训练，但它在多模态提示方面展现出新兴的零样本能力。最后，虽然当前优先考虑高层语义而非细粒度细节，但GAP3D表明，通过扩散对齐可以部分弥合VLM与图像编码器特征空间之间的表示差距，为通过生成对齐到密集嵌入空间实现基础模型的模块化整合迈出了第一步。

## Abstract
Recent approaches integrating vision-language models (VLMs) as prompt encoders for generative model conditioning typically rely on expensive end-to-end training or map features to compressed representations, discarding the dense spatial structure required for geometry-aware tasks like 3D asset generation. To address this, we propose GAP3D, a modular, diffusion-based approach that aligns VLM-generated latents directly to the complete, patch-level feature space of a pre-trained image encoder, enabling a frozen downstream generative model to utilize a VLM as prompt encoder while maintaining a spatially structured conditioning signal. Evaluated on 3D asset generation, our method bypasses the need for large-scale 3D data by training mainly on general-domain image-text pairs. It also exhibits emergent zero-shot capabilities for multimodal prompts, despite being trained exclusively on text input. Finally, while currently prioritizing high-level semantics over fine-grained detail, GAP3D demonstrates that the representation gap between VLM and image-encoder feature spaces can be partially bridged through diffusion-based alignment, taking the first steps towards a modular integration of foundation models through generative alignment to dense embedding spaces.

- **核心问题**：现有生成模型通常使用CLIP、T5等文本编码器作为条件编码器，但受限于输入长度、缺乏空间理解，在处理长文本、属性绑定和复杂结构时表现不佳。虽然VLM（视觉语言模型）具备强大的多模态推理能力，但将其集成到生成管道中面临两难：端到端训练成本高、需大量域内数据；模块化方法通常将VLM特征映射到压缩的嵌入（如池化后的CLIP向量或少量文本token），丢失了用于几何感知任务（如3D资产生成）所必需的**密集空间结构**。

## 方法论：核心思想与关键技术细节

- **核心思想**：采用**扩散对齐**（基于流匹配）将VLM生成的潜在特征映射到DINOv2图像编码器的完整高维特征空间（CLS、寄存器、所有patch嵌入），从而将文本-to-3D任务转化为“伪图像-to-3D”任务，使冻结的3D生成模型（TRELLIS）利用其原本强大的图像条件分支进行生成。
- **关键技术细节**：
  1. **VLM编码阶段**：冻结VLM（Qwen2.5-VL-3B-Instruct），在输入文本中拼接可学习的软token（s个），提取这些软token对应的隐藏状态作为条件信号C ∈ ℝ^{s × D_vlm}。VLM参数冻结，仅优化软token。
  2. **目标表示空间**：选择DINOv2（ViT-L/14, register variant）作为目标图像编码器，其输出包括：
     - 密集的2D patch特征网格 x_p ∈ ℝ^{h×w×1024}（37×37网格，即1369个token）
     - 全局CLS token x_cls ∈ ℝ^{1024}
     - 4个寄存器token x_reg ∈ ℝ^{4×1024}
  3. **生成架构**：基于Lumina-Next的DiT（扩散Transformer），修改输入嵌入层：
     - 对patch网格使用线性嵌入 + 2D RoPE。
     - 对CLS和寄存器token使用独立线性投影 + 可学习位置嵌入 + 1D RoPE（旋转因子为恒等）。
     - VLM条件C通过交叉注意力注入网络。
  4. **训练目标**：使用**Rectified Flow**，定义三个同步流路径（patch、CLS、寄存器），共享时间步t。从高斯噪声出发，线性插值到真实DINOv2嵌入。损失为各分量流匹配损失的加权和：
     - L(θ) = E_{t,x0,x1} [ Σλ_k ||v^θ_k(xt,t,C) - (x1^k - x0^k)||² ]，其中λ_p=0.4, λ_cls=0.3, λ_reg=0.3。
  5. **两阶段训练**：
     - 阶段1（预训练）：在通用图像-文本对（约3100万张）上训练，学习通用映射。
     - 阶段2（微调）：在3D模型渲染图像（约6万张）上微调，以适应3D渲染的分布偏移（干净背景、物体中心、均匀光照等）。

## 主要结论与发现

1. **生成对齐可行**：通过流匹配可以将VLM潜在表示对齐到DINOv2的完整patch级嵌入空间（1369个token），余弦相似度达0.67（微调后在Toys4K上），表明高维稠密空间的对齐是学习可及的。
2. **域适应至关重要**：预训练模型在自然图像上表现更好，但直接用于3D生成会产生严重背景伪影和结构扭曲。领域微调显著提升3D生成质量（FD从41.90降至24.20，CLIP Score从27.51升至29.26），而仅靠提示工程无法弥补分布偏移。
3. **零样本多模态能力**：尽管仅使用文本训练，GAP3D可接受图像+文本的多模态输入，并执行语义级的编辑（如改色、改材质），但无法精细保留输入几何，主要工作模式是“语义翻译器”。
4. **局限性**：对细粒度几何和低层视觉细节的捕捉不足；CLS嵌入的对齐优于patch嵌入；多模态编辑更偏向高层语义而非结构保留；域适应导致在通用数据上的灾难性遗忘。

## 优点

- **模块化设计**：VLM、对齐模块、下游3D生成模型均可独立冻结/替换，无需端到端重训，降低了计算和数据需求。
- **创新的对齐目标空间**：首次将扩散对齐扩展到完整patch级空间（1369个token），而非压缩的池化特征，为需要密集空间信息的任务（如3D、分割）提供了新思路。
- **高质量的域适应策略**：两阶段训练（通用预训练+领域微调）有效缓解了分布偏移，且微调数据仅需6万张渲染图，数据效率高。
- **严格遵循下游评价协议**：3D生成评估完全复现TRELLIS论文的设置（渲染视角、特征提取器、指标），确保了对比的公正性。

## 不足与局限

- **细粒度细节缺失**：生成的嵌入在高层语义上对齐较好，但无法准确重建颜色、形状等低层属性（表现为检索中常返回形状相似但颜色不同的物体，3D生成中指定颜色/形状缺失），作者归因于软token容量不足和缺乏像素级重构预训练。
- **灾难性遗忘**：微调后模型在通用域（MS-COCO）上对齐性能大幅下降，未混合通用数据进行微调。
- **计算资源需求高**：需要128 GPU进行预训练，可能限制广泛复现；消融实验不完整（未报告软token数量、损失权重等超参数的影响）。
- **多模态编辑能力粗糙**：仅能实现语义级变化，无法保持输入图像几何细节，更接近“基于语义的重新生成”而非“编辑”。
- **客观公平性讨论**：与BLIP3-o的比较因模型规模、训练数据、目标空间差异显著而不完全公平；未与其他近期VLM-to-3D方法（如使用LLM驱动的3D生成）作对比。
- **实验覆盖有限**：仅在两个数据集上评估3D生成，且Toys4K为相对简单的物体级数据集；未在更多样化、复杂场景（如大场景、部件级生成）上测试；缺少人工评估。
