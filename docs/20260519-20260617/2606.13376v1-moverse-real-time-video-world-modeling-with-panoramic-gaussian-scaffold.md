## 摘要
我们提出MoVerse，一种实时视频世界模型，能够从单张窄视场图像创建可交互导航的场景。这个设定具有挑战性，因为输入只观察到环境的一小部分，而交互式漫游需要完整的周围世界、持久的几何结构、可控的相机运动以及时间一致的高保真观测。MoVerse通过将世界构建与观测渲染分离来解决这个问题。它首先利用拓扑感知扩散将输入扩展为重力对齐的360°全景图，在3D推理之前填补缺失的视场。然后，使用全景几何感知残差预测将全景图提升为持久3D高斯脚手架，生成密集且可直接渲染的空间记忆。最后，一个高斯条件视频渲染器将沿用户指定相机轨迹的脚手架渲染转换为逼真的视频。为了使这个渲染器适用于交互，我们训练了一个双向扩散教师模型用于高质量条件渲染，并将其蒸馏到一个因果自回归学生模型用于有界延迟流式处理。这种设计结合了显式3D表示的可控性和长程一致性，以及生成视频模型的感知质量。MoVerse在单个NVIDIA RTX 4090 GPU上支持8 FPS的实时场景漫游，展示了从单张图像创建交互式视频输出的实用路径。

## Abstract
We present MoVerse, a real-time video world model that creates an interactively navigable scene from a single narrow-field-of-view image. This setting is challenging because the input observes only a small fraction of the environment, while interactive roaming requires a complete surrounding world, persistent geometry, controllable camera motion, and temporally coherent high-fidelity observations. MoVerse addresses this problem by separating world construction from observation rendering. It first expands the input into a gravity-aligned 360$^\circ$ panorama with topology-aware diffusion, closing the missing field of view before 3D reasoning. It then lifts the panorama into a persistent 3D Gaussian scaffold using panoramic geometry-aware residual prediction, yielding a dense and directly renderable spatial memory. Finally, a Gaussian-conditioned video renderer translates scaffold renderings along user-specified camera trajectories into photorealistic video. To make this renderer practical for interaction, we train a bidirectional diffusion teacher for high-quality conditional rendering and distill it into a causal autoregressive student for bounded-latency streaming. This design combines the controllability and long-range consistency of explicit 3D representations with the perceptual quality of generative video models. MoVerse supports real-time scene roaming at 8~FPS on a single NVIDIA RTX~4090 GPU, demonstrating a practical path toward single-image world creation with interactive video output.

- **现有方法的不足**：
  - 显式3D方法（点云、网格、3D高斯）可提供持久空间记忆和实时渲染，但从单张NFOV图像直接重建时，大量内容需要推断，容易出现空洞、浮动伪影、深度错误。
  - 隐式视频世界模型（Genie、Raven等）能生成视觉上强的视频，但长程几何靠隐式记忆维持，容易漂移、身份变化、边界不连续。
  - 混合系统（几何条件+生成渲染）通常存在几何锚点稀疏或渲染器过慢的问题。
- **整体含义**：提出一种**分离世界构建与观测渲染**的三阶段方案，将显式几何的可控性和生成模型的视觉质量结合起来，实现**从单图到实时交互视频世界的实用路径**。

## 2. 论文提出的方法论

### 2.1 核心思想

> 将世界构建（Stage I + II）与观测渲染（Stage III）解耦。前两个阶段离线构建持久3D高斯脚手架，第三阶段在交互时沿用户轨迹渲染脚手架并翻译为高保真视频。

### 2.2 关键技术细节

#### Stage I: 全景生成（NFOV图像 → 重力对齐的360° ERP全景）
- **差异化自动调平**：预测密集2D对应场，通过可微分的软argmin求解器强制为刚体球面旋转，实现输入图像的姿态规范化，消除俯仰和翻滚。
- **拓扑感知扩散完成**：
  - 采用**圆形潜编码**（卷积padding沿方位角环绕），保证潜空间水平闭合。
  - 训练**平移等变性损失**：对输入进行水平循环移位后，预测的噪声也应一致移位，消除固定拼接边界。
  - **训练数据（Horizon360）**：对原始全景进行重力规范化（旋转使地平线与ERP赤道对齐），并中心化输入视角（yaw-centered），避免已知区域靠近周期边界。

#### Stage II: 高斯场景生成（全景 → 直接可渲染的3DGS脚手架）
- **全景高斯初始化**：
  - 从ERP深度图球面反投影得到高斯中心。
  - **纬度感知尺度**：初始尺度∝ D·cos(ϕ)，避免高纬度退化。
- **深度引导残差预测**：从全景RGB+逆深度提取多尺度特征，预测残余高斯属性（角度偏移、逆深度偏移、尺度、四元数、颜色、不透明度）。采用**零初始化**，训练前保持物理初始化，学习局部修正。
- **角度-逆深度空间残差组合**：位置偏移在(Δθ, Δϕ, Δz)空间中预测，再球面反投影到3D中心，保持与ERP网格一致性和水平闭合。
- **训练损失**：目标视角渲染损失(L1+LPIPS+SSIM) + 源视角重建损失 + 逆深度损失 + 残差正则化项。

#### Stage III: 高斯条件视频渲染（脚手架渲染 → 高保真视频）
- **双向扩散教师**：基于Wan2.1-T2V-1.3B初始化，将脚手架渲染视频VAE潜码与目标视频潜码通道拼接，采用**共享RoPE位置编码**，鼓励模型将条件视为对齐的渲染指导。训练流匹配目标。
- **因果自回归学生**：通过Self Forcing和DMD蒸馏，学生每次生成一个潜块（对应4帧），条件于当前脚手架渲染块和历史KV缓存。使用**MemRoPE风格的局部KV缓存**（1个sink帧+1个EMA记忆token+3帧滑动窗口），保证流式低延迟。
- **训练数据混合物**：三种来源逐步逼近部署场景：
  1. 真实视频+单视点深度重投影（学习基本修补）。
  2. 真实视频+稀疏关键帧高斯脚手架渲染（学习高斯渲染伪影修复）。
  3. 全景到高斯脚手架渲染（匹配最终接口），来自UE渲染、Matterport3D、Polyhaven等。
- **实时推理**：两步去噪调度，TAEHV解码器，8 FPS on RTX 4090。
