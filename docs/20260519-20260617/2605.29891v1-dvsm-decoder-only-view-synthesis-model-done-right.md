## 摘要
近期的大规模视角合成模型（LVSMs）倡导采用编码器-解码器架构，将重建和渲染分离到不同的网络中。我们重新审视了这种设计。通过控制实验，我们表明仅解码器架构（隐式地将场景表示为KV缓存）在相同渲染复杂度下使用更少的参数，其性能优于编码器-解码器变体。进一步分析表明，在颜色输入重建网络和仅相机渲染网络之间共享权重，能更好地在同一视点对齐它们的特征，从而促进图像合成。基于这一发现，我们的模型（称为DVSM）进一步融合了基础模型先验和分阶段块大小调整，以改进效率与质量的权衡。我们的结果在多个基准测试中为新颖视角合成设立了新的最优水平，在某些情况下甚至超过了在密集输入视角下针对每个场景优化的3DGS。

## Abstract
Recent Large View Synthesis Models (LVSMs) advocate an encoder-decoder architecture that separates reconstruction and rendering into distinct networks. We re-examine this design. Through controlled experiments, we show that a decoder-only architecture, which represents scenes implicitly as a KV-cache, outperforms encoder-decoder variants while using fewer parameters at identical rendering complexity. Further analysis shows that sharing weights between the color-input reconstruction network and the camera-only rendering network better aligns their features at the same viewpoint, facilitating image synthesis. Building on this finding, our model, dubbed DVSM, further incorporates foundation model priors and stage-wise patch sizing for an improved efficiency-quality tradeoff. Our results establish a new state of the art for novel-view synthesis across multiple benchmarks, in some cases even outperforming per-scene-optimized 3DGS under dense input views.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：现有的大型视角合成模型（LVSMs）普遍采用编码器-解码器架构，将场景重建和渲染分为两个独立网络。这种分离导致参数冗余、特征不对齐，且渲染效率受限。作者质疑这种设计是否最优。
- **研究动机**：受经典可微渲染方法（如NeRF、3DGS）启发，它们共享重建与渲染网络（通过可微渲染器同时完成场景优化和渲染）。作者希望探索一种更简洁、高效的仅解码器架构，使重建与渲染本质上是同一网络的不同阶段。
- **整体含义**：提出DVSM（Decoder-only View Synthesis Model Done Right），验证在视图合成中完全共享重建和渲染权重（包括输入嵌入层）能带来显著的性能提升，同时参数量更少、渲染速度更快。该工作为大规模视图合成提供了一种新的高效范式。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想
- **场景隐式表示为KV缓存**：在重建阶段，将所有视角的图像和相机参数编码为token，通过包含自注意力、交叉注意力和MLP的Transformer解码器处理，并缓存所有交叉注意力层的K和V。渲染阶段，仅用查询相机的Plücker坐标生成查询token，通过同一解码器交叉注意力层从缓存的KV中检索场景信息，最后解码为图像。
- **完全权重共享**：重建阶段和渲染阶段使用完全相同的权重，包括输入patch嵌入层、自注意力、交叉注意力、MLP所有层。这是最关键的设计。
- **基础模型先验注入**：在重建阶段，将预训练的DINOv3特征（经线性投影）加性注入到输入token中，以提供更丰富的视觉先验，提升渲染质量且不增加渲染计算量。
- **分阶段块大小（Stage-wise Patch Sizing）**：在重建和渲染阶段使用不同的patch尺寸（例如重建使用更小patch以保留细节，渲染使用更大patch以提高速度），通过调整输入图像尺寸来共享同一个patch嵌入层。

### 2.2 关键技术细节
- **输入处理**：
  - 重建阶段token：`X_recon = LayerNorm(PE_ray(Plücker(P,K)) + PE_rgb(I) + Prior(I))`，其中Prior基于DINOv3。
  - 渲染阶段token：`X_rend = LayerNorm(PE_ray(Plücker(P',K')))`，只有相机信息。
- **Transformer结构**：L层重复块，每块包含一个intra-view self-attention、一个cross-view attention（重建阶段缓存KV，渲染阶段从缓存查询），以及两个MLP。采用残差连接和QK归一化。
- **损失函数**：`L = MSE(I_hat, I') + λ · Perceptual(I_hat, I')`，MSE和感知损失。
- **场景表示**：KV缓存本身即隐式场景表示，无需显式3D几何。

## 6. 论文的主要结论与发现

- **仅解码器架构优于编码器-解码器**：在相同渲染复杂度下，DVSM用更少参数（164M vs 308M）实现更高PSNR（24.60 vs 23.19）。完全权重共享是关键。

- **基础模型先验（DINOv3）带来显著提升**：在Re10K上PSNR提升约1.6 dB。
- **分阶段patch尺寸提供更好的效率-质量权衡**：重建用小patch、渲染用大patch可在几乎不改重建时间下提升渲染质量。
- **在多个基准上达到新SOTA**：

  - DL3DV 64视图：PSNR 29.71，与3DGS全视图优化结果（29.82）接近。
  - 零样本泛化：在MipNerf360等数据集上大幅超越LVSM和LaCT。
  - ScanNet++：在非受控视频上远优于3DGS。
- **局限性**：渲染速度仍不足以实时（15-40 FPS），大输入视图GPU内存溢出，泛化性受限于训练数据集规模和多样性。

## 8. 不足与局限

- **渲染速度仍有限**：即使使用ps16，FPS只有15-40，远不如3DGS（>50）。无法实时。
