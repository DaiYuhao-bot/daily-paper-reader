## 摘要
文本和图像条件约束的3D模型现在能生成令人信服的资产，但它们仍然缺乏对物体应占据或避免的空间的直接控制。在创作过程中，这种空间意图通常在生成开始前就已明确：一把椅子应贴合坐姿包络、一个道具应为运动留出间隙、或一个部件应露出接触面。提示和图像视图难以承载此类约束，亟需显式控制界面。我们提出Arbor，一种可训练的附加模块，用于文本条件约束的潜变量3D生成。Arbor将约束网格作为原生3D控制界面引入。该界面使用应存在几何体的壳体区域、应保持空的规避区域以及物体应接触的触摸区域。与补全或整体物体支架控制不同，这些网格并非目标证据，而是局部类型化要求，可包含不应出现表面的区域。Arbor通过将约束网格转换为令牌并在冻结去噪器内学习路由附加模块，将此信号保持为几何形式。因此，每个潜变量区域能够接收对其空间位置至关重要的约束部分。我们在自动和艺术家策划的控制基准上评估Arbor，涉及壳体、规避和触摸约束，并将指标趋势与用户偏好研究进行比较。即使没有专用的合规损失，Arbor也能在保持固定约束下的物体质量和多样性的同时，提高约束服从性。

## Abstract
Text and image conditioned 3D models now generate convincing assets, but they still offer little direct control over the space an object should occupy or avoid. In authoring, this spatial intent is often known before generation starts. A chair should fit a seating envelope, a prop should leave clearance for motion, or a part should expose a contact surface. Prompts and image views are poor carriers for such constraints, requiring the need for an explicit control interface.   We present Arbor, a trainable attachment for text conditioned latent 3D generation. Arbor introduces constraint meshes as a native 3D control interface. The interface uses hull regions where geometry should exist, avoidance regions that should remain empty, and touch regions the object should contact. Unlike completion or whole object scaffold control, these meshes are not target evidence. They are local typed requirements and can include regions where no surface should appear. Arbor keeps this signal as geometry by converting constraint meshes into tokens and learning a routed attachment inside a frozen denoiser. Each latent region can therefore receive the part of the constraint that matters for its spatial location.   We evaluate Arbor on automatic and artist curated control benchmarks with hull, avoidance, and touch constraints, and compare the metric trends to a user preference study. Even without dedicated compliance losses, Arbor improves constraint obedience while preserving object quality and variation under fixed constraints.

## 1. 核心问题与整体含义

- **研究动机**：当前文本/图像条件3D生成模型虽然能生成令人信服的资产，但缺乏对物体应占据或回避的空间的直接控制。艺术家在创作前往往已明确空间意图（如椅子应贴合坐姿包络、道具应为运动留间隙、部件应露出接触面），但提示词和图像视图难以承载此类精细空间约束，导致生成结果像“老虎机”一样随机，需要大量手动清理。
- **整体含义**：Arbor首次将显式几何约束（约束网格）作为原生3D控制接口引入潜空间生成模型，让艺术家能够通过定义壳体（hull）、规避（avoidance）、接触（touch）三类区域来引导生成，从而在保持生成质量和多样性的前提下显著提升约束服从性。这项工作填补了3D生成中“空间意图控制”的空白。

## 2. 方法论

### 核心思想
- 在冻结的TRELLIS文本条件3D生成器（稀疏结构阶段）中插入可训练的路由附加模块，将用户提供的约束网格转换为几何令牌，并按空间位置路由到对应潜变量区域，使每个潜变量区域只接收对其空间位置至关重要的约束部分。

### 关键技术细节
- **约束定义**：三类约束网格——壳体（hull，几何应存在的区域）、规避（avoidance，应保持空的区域）、接触（touch，物体应接触的表面）。
- **约束编码**：利用冻结的TRELLIS.2（T2）编码器，将约束网格体素化为512³的OVoxel，将材料通道重用于编码约束类型（二进制标志：c3壳体、c4规避、c5触），并保留法线通道。编码后得到稀疏32³几何令牌（每个32维）。
- **投影与位置嵌入**：将几何令牌通过可学习投影映射到TRELLIS.1（T1）模型宽度，并添加可学习的3D位置嵌入。
- **路由机制**：将T1的16³潜变量格点划分为64个4×4×4的查询组。每个查询组通过其8个角点作为锚点，从全局几何令牌中选取K=2048个最近的令牌作为局部上下文；同时使用96个可学习摘要令牌捕获全局信息。局部上下文与全局摘要拼接后用于注意力更新。
- **适配器注入**：在每个T1 transformer块中，在冻结的文本交叉注意力之后、前馈网络之前插入一个几何接地分支。该分支使用当前查询组隐藏状态作为query，对路由上下文进行cross-attention，输出残差更新（初始化为零），再与前馈网络输出相加。
- **训练策略**：仅训练几何相关模块（投影、位置嵌入、全局摘要、接地适配器等），T1的自注意力、文本交叉注意力、前馈网络权重冻结。优化目标采用T1的流匹配损失，无显式约束合规损失。文本和几何条件独立丢弃以实现无分类器引导。

### 公式与算法流程（文字说明）
1. 输入文本提示y和约束对象C（包含壳体、规避、接触网格）。
2. 使用T2编码器将C转换为几何令牌c_geo（形状+信号）。
3. 将c_geo投影到T1宽度，添加位置嵌入。
4. 路由：按查询组从c_geo中选取最近K个令牌，并生成全局摘要。
5. 在T1每个block中：执行冻结的文本交叉注意力 → 执行几何交叉注意力（查询为当前隐藏状态，键/值为路由上下文） → 残差更新 → 执行冻结的FFN。
6. 解码生成最终3D资产。

## 6. 主要结论与发现

- Arbor在约束遵循度上显著优于所有非Arbor基线（包括无几何条件的TRELLIS、采样时引导的SpaceControl、训练后引导的Spice-E、推理时梯度修正），同时保持合理的物体质量和多样性。
- 在用户研究中，Arbor家族（合并三个变体）以59.2%的胜率最受青睐，单个Arbor以45.9%领先其他方法。
- 约束遵循度提升并非以生成质量下降为代价：Volume Match和MV-CLIP分数与基线相当或略优。
- 路由消融表明局部路由是主要机制（保留84.3% IoU），全局摘要提供边际增益；信号流（类型标志）和形状流都很关键。
- 固定约束变化实验显示Arbor在控制下仍保持高变化性（0.740），远高于图像条件模型（SPAR3D 0.141、Hunyuan3D-Omni 0.526）。
- 即使没有显式合规损失，Arbor也能通过学习路由附件有效整合约束。

## 7. 优点

1. **显式几何控制接口**：引入壳体、规避、接触三类类型化约束网格，直观易用，且可以灵活组合。
2. **冻结骨干、轻量附加**：仅训练路由附件，不修改TRELLIS骨干，保持基础先验完整，训练成本低（约5万物体、80小时8GPU）。
3. **路由机制高效**：通过空间分组和Top-K选择，将稠密几何令牌压缩到固定预算（每个组2048个），同时通过全局摘要保留整体信息。
4. **无需额外损失函数**：仅使用TRELLIS原有流匹配损失，无需显式合规损失即可实现约束遵循，表明方法自然适应生成先验。
5. **广泛消融和用户研究**：验证了路由、信号流、形状流等各组件的必要性，并通过用户偏好确认自动指标趋势与人类判断一致。
6. **稳健全面**：在自动和手动（分布外）基准上均表现良好，且能处理移动、缩放、旋转等连续约束变化。

## 8. 不足与局限

1. **仅作用于稀疏结构阶段**：Arbor只控制TRELLIS稀疏结构生成，后续的SLAT精炼和解码阶段不受直接控制，可能丢失局部细粒度细节。
2. **约束与提示可能存在冲突**：非常小的壳体、弱语义线索或激进的规避区域会迫使模型在遵循提示和约束之间妥协，导致某些样本质量下降。
3. **语义功能缺失**：约束区域仅携带几何和类型信号，但无语义功能（如“座位”区域可给出体积和方向，但不能保证生成体积确实被用作座位，尤其当提示与约束矛盾时）。语义变体（Arbor Semantics）尚未超越路由几何路径。
4. **基准异质性**：对比方法来自不同线（编辑、补全、采样时引导、图像条件重建），并非统一“文本+几何约束”框架，公平比较存在难度。模态特异方法仅在变化表中出现。
5. **训练数据局限**：训练集仅约5万物体（TRELLIS原始训练的10%），多为简单场景；在更复杂或极端约束下泛化性未充分验证。
6. **手动基准规模小**：手动分割仅32个样本，虽为分布外但统计意义有限。用户研究404次选择，参与者27人，规模中等。
7. **未与更多最新控制方法对比**：例如未与SK-Adapter、Coin3D、Hunyuan3D-Omni（在控制生成表中未出现）等在同等条件下对比。
8. **透明度限制**：未公开代码和数据包（计划未来发布），可复现性受限。
