## 摘要
从新相机视角重新渲染现有视频，要求输出遵循预设的相机轨迹，同时保持原始场景在每一帧中的外观和动态。现有方法依赖于逐帧姿态嵌入、带噪声的点云渲染或隐式学习的对应关系，但均未提供源像素与目标像素之间的显式、时间连续连接。我们提出Track2View，该方法以成对3D点轨迹（即投影到源视图和目标视图中的场景点稀疏轨迹）为条件，驱动视频扩散变换器。这些轨迹通过构造提供显式的时空对应关系，且时间连续，编码了内容应在何时何地出现。Track2View的核心是一个双视图轨迹调节器，通过无参数几何操作和学习到的时间聚合，将视觉上下文从源视图传递到目标视图，确保对任意相机轨迹的泛化能力，而无需记忆特定运动。我们进一步引入一个数据整理流程，通过在时间上拼接的多相机视图对上运行3D点追踪器，提取一对一的轨迹对应关系。在包含静态和动态场景的400个视频基准测试中，Track2View在视觉质量、视图同步和相机准确性方面均达到最先进水平，相对于领先基线，旋转误差降低30-65%，平移误差降低61-72%。项目页面见：https://qjizhi.github.io/track2view

## Abstract
Re-rendering an existing video from a novel camera viewpoint requires the output to follow the prescribed camera trajectory while preserving the appearance and dynamics of the original scene across every frame. Existing methods rely on per-frame pose embeddings, noisy point-cloud renderings, or implicit learned correspondences, none of which provides an explicit, temporally continuous link between source and target pixels. We propose Track2View, which conditions a video diffusion transformer on paired 3D point tracks: sparse trajectories of scene points projected into both the source and target camera views. These tracks provide explicit spatiotemporal correspondences that are temporally continuous by construction, encoding what content should appear where and when. At the core of Track2View is a dual-view track conditioner that transfers visual context from source to target view through parameter-free geometric operations and learned temporal aggregation, ensuring generalization to arbitrary camera trajectories without memorizing specific motions. We further introduce a data curation pipeline that extracts one-to-one track correspondences by running a 3D point tracker on temporally concatenated multi-camera view pairs. On a 400-video benchmark spanning static and dynamic scenes, Track2View achieves state-of-the-art results across visual quality, view synchronization, and camera accuracy, reducing rotation error by 30-65% and translation error by 61-72% relative to leading baselines. # Track2View 论文总结

## 1. 核心问题与整体含义（研究动机与背景）
- **问题**：从单段源视频出发，根据用户指定的目标相机轨迹重新渲染出新视角视频，要求输出在每一帧遵循相机路径，同时保持原场景的外观与动态，即实现**4D一致性**（时空同时连贯）。现有方法存在三种局限：
  - **逐帧姿态嵌入**（如Trajectory Attention、TrajectoryCrafter）：假设源相机静止，当源相机移动时降级为图像到视频模式，且逐帧处理无时间连续性约束。
  - **点云渲染引导**（如Gen3C）：依赖单目深度估计，噪声大、逐帧渲染导致遮挡区域闪烁。
  - **隐式学习对应**（如ReCamMaster）：仅通过注意力机制隐式建模几何，大视角变化下相机精度下降。
- **核心挑战**：缺少一种既**几何显式**又**时间连续**的源-目标像素对应信号。
- **整体含义**：Track2View首次引入**成对3D点轨迹**作为显式条件，通过构造提供时空连续对应，填补了上述空白，实现高质量、高一致性的相机可控视频再渲染。

## 2. 方法论
- **核心思想**：利用3D点轨迹（场景点在源和目标相机序列下的稀疏2D投影轨迹）作为条件信号，送入视频扩散Transformer。轨迹定义在全局坐标系，不受源相机运动影响，本身时间连续；同时提供明确运动向量，指示每个源像素在目标视频中的位置。
- **关键流程**：
  1. **轨迹提取**：用SpatialTrackerV2从源视频联合估计相机姿态和3D点轨迹，再将轨迹按目标相机参数重投影得到目标轨迹。
  2. **双视图轨迹调节器**：
     - 将轨迹投影为2D坐标和深度，从源视频token中采样得到稀疏轨迹特征。
     - 添加傅里叶位置编码，经MLP精炼。
     - 用L层（默认8层）Transformer编码器沿时间维度聚合轨迹特征，传递可见帧到不可见帧的视觉上下文。
     - 注入视点特定的逆深度编码（源/目标视点用各自的深度，共享时间特征）。
     - 通过双线性散射将轨迹特征映射回密集空间网格，生成源和目标轨迹token。
  3. **模型架构**：基于WAN-2.1预训练DiT，将源与目标视频token沿帧维度拼接，加上轨迹token后经DiT去噪。只对DiT使用LoRA微调，保持其他参数冻结。
  4. **训练数据准备**：利用MultiCamVideo合成数据集，将源相机A的视频反转后与目标相机B的视频拼接（首帧相同），形成一个161帧序列。用SpatialTrackerV2从该序列中追踪一致的3D点，从而自然得到一对一对应轨迹。
- **公式/算法**（文字说明）：
  - 轨迹特征采样：`τ̂ = bilinear_sample(ν_src, T_xy_src)`
  - 特征精炼：`τ̂ ← τ̂ + PE_xy + MLP(τ̂ + PE_xy)`
  - 时间聚合：`τ̄ = TemporalAgg(τ̂)`（Transformer）
  - 深度编码：`v_src = τ̄ + PE_z(T_z_src)`, `v_tgt = τ̄ + PE_z(T_z_tgt)`
  - 散射：`τ_src = Scatter(v_src, T_xy_src)` 等
  - 损失：只对目标部分计算标准流匹配损失。

## 7. 优点
- **显式几何对应**：3D点轨迹提供源-目标像素间的明确运动向量，无需学习隐式对应，避免了噪声渲染和注意力负担。
- **时间连续**：轨迹本身连续，克服逐帧处理的闪烁问题。
- **泛化能力**：双视图调节器中所有几何操作（采样、散射）无参数，仅时间聚合和特征投影可学习，因此能泛化到任意未见过的相机轨迹。
- **数据准备巧妙**：通过反转+拼接多相机视频，自然获得一对一对应轨迹，无需显式匹配。
- **高效微调**：仅用LoRA适配基模型，保持预训练能力，训练成本相对可控（576GPU小时）。

## 8. 不足与局限
- **依赖上游点追踪器质量**：如果场景中追踪失败（如严重遮挡、快速运动、缺乏纹理），可能导致轨迹质量差，进而影响生成效果。论文未对这种失败案例进行专门分析。
- **训练数据全为合成数据**（Unreal Engine 5），虽能泛化到真实视频（RealEstate10K、MiraData），但**极端分布外场景**（如极端光照、非刚性变形、运动模糊严重）的行为尚未验证。
- **误差累积**：长序列（81帧）下相机误差自然增长，虽仍优于基线，但绝对误差值（TransErr 0.818）仍有提升空间。
- **未考虑置信度**：所有轨迹等权使用，若引入置信度加权可能进一步提升鲁棒性，论文也承认这是未来方向。
- **推理速度**：基于DiT+ LoRA + 50步去噪，推理速度可能较慢，论文未报告FPS等实际效率指标。
