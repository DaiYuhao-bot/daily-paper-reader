## 摘要
文本到3D生成降低了3D内容创作的门槛，但仅靠文本是一种弱接口，难以指定空间意图：部件应放置在哪里、它们之间的关系如何、以及对象在3D中应如何组织。我们提出HandMade，一种结合VR 3D草图和语言的工作流程，用于开放域3D资产生成。HandMade将粗粒度的、带部分标签的3D草图视为现有生成模型的空间提示，而非直接重构的不完整几何体。它将分段式VR笔画转换为多视图部件引导和结构化提示，使用户能够通过3D草图指定对象布局和部件关系，同时使用语言指定身份、材质、风格和局部细节。技术评估表明，在20个多样化示例上，HandMade比仅文本和基于草图的基线方法更好地保留了用户创作的空间支架。一项包含八名参与者的用户研究描述了用户如何在初始创作和后续修改中利用3D草图进行空间布局以及利用语言指定身份、材质和细节。HandMade贡献了一种用于空间引导3D创作的交互范式和从界面到生成的管道。

## Abstract
Text-to-3D generation lowers the barrier to 3D content creation, but text alone is a weak interface for specifying spatial intent: where parts should be placed, how they relate, and how an object should be organized in 3D. We present HandMade, a workflow that combines VR 3D sketching and language for open-domain 3D asset generation. HandMade treats coarse, part-labeled 3D sketches not as incomplete geometry to reconstruct directly, but as spatial prompts for existing generative models. It converts segmented VR strokes into multi-view part guidance and structured prompts, allowing users to specify object layout and part relationships through 3D sketching while using language for identity, material, style, and local details. A technical evaluation shows that HandMade better preserves user-authored spatial scaffolds than text-only and sketch-based baselines on 20 varied examples. A user study with eight participants characterizes how users make use of 3D sketching for spatial layout and language for identity, materials, and details across initial authoring and subsequent revision. HandMade contributes an interaction paradigm and interface-to-generation pipeline for spatially guided 3D creation.

### 2. 论文提出的方法论
- **核心思想**：把用户绘制的带部件标签的粗VR 3D草图视为空间引导信号，而不是精确几何约束；语言负责语义、材质、风格等细节。两者结合后，通过多步骤管道转化为3D模型。
- **关键技术细节**：
  1. **输入**：用户通过VR手绘三维笔画，每个部件用系统划分的不同颜色通道表示；用户为每个部件命名并可选写描述，同时提供整体对象文本描述。
  2. **多视图部件引导**：将整个草图归一化到单位立方体，从四个固定相机视角（前、右、后、左，俯仰角0°）渲染成2×2网格的部件颜色引导图，背景白色。
  3. **结构化提示构建**：提示包含三部分：输出规格（要求生成干净的四视图渲染）、颜色到部件名称的映射、用户的部件及对象描述。提示中明确说明引导图是粗略VR手绘的近似表达。
  4. **多视图图像合成**：将引导图和文本提示输入图像生成模型（本文使用ChatGPT Images 2.0），生成对象干净的4视图图像。
  5. **3D形状合成**：将4视图图像分割后输入多视图图像到3D重建模型（本文使用Hunyuan3D-2mv），输出带纹理的网格（GLTF格式），包含网格减面（10万面）和纹理绘制步骤。
- **算法流程**：VR草图 → 归一化 → 四视图渲染 → 构造提示 → 图像生成 → 分割视图 → 3D重建 → 纹理网格。

### 6. 论文的主要结论与发现
- HandMade在所有几何对齐指标上均优于所有基线，尤其比文本-only和单视图更好的保留了用户草图中的空间布局（Recall@0.05达0.792，F-score 0.727）。
- 消融实验证明多视图引导至关重要，单视图效果差于多视图，文本-only效果最差。
- 用户研究发现：用户主要用3D草图表达部件位置、关系和大致形状，用语言描述身份、材质、风格和局部细节；修订主要发生在非草图阶段（修改文本或重新生成），说明了草图锚定后调整的便利性。
- 工作流中的中间检查点（多视图图像、网格）帮助用户定位并局部修复不匹配，提高用户控制感。

### 7. 优点
- **创新性交互范式**：首次将带部件标签的粗VR 3D草图作为“空间提示”用于开放域生成，而非传统直接重建或精确控制。
- **模块化设计**：利用现有强大的通用图像/3D生成模型，无需训练新模型，易于适配未来更好的生成器。
- **设计目标明确**：支持不精确输入、保留部件意图、分离空间与语义、开放域、支持修订，贴合早期创意阶段需求。
- **工作流实用**：用户可迭代调整，中间结果可用作检查点，降低试错成本。

### 8. 不足与局限
- **输入精度受限**：VR手绘本身受手部追踪噪声、深度控制困难、绘画技能限制，草图可能无法表达精确平面、对称、连接等关系。
- **管道依赖第三方模型**：图像生成和3D重建模型可能忽略引导图、误解比例、产生视图不一致或丢失细小结构，影响最终结果。
- **技术评估范围窄**：仅20个示例，且只测量几何对齐，未评估美学、语义一致性、用户满意度等；用户研究样本仅8人，代表性不足。
- **交互负担**：当前原型需要研究者介入阶段转换，未全自动化；佩戴VR头显和键盘操作有学习成本；部分用户反馈存在控制、视角切换、头晕等体验问题。
- **无法控制关系约束**：当前无法显式指定部件间“接触”、“分离”、“平行”等空间约束，仅靠文本描述难以准确传播到生成阶段。
- **可读性/编辑性**：输出网格拓扑未优化，可能难以直接用于下游精细编辑；未支持部件锁定、选择性再生等高级功能。
