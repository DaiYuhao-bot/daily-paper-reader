## 摘要
自动驾驶的生成式世界模型面临两个未解决的对立问题：异质控制注入（自由形式语言、高清地图、轨迹和相机姿态存在于不兼容的表示空间中）以及事后交叉视图融合（每个相机的潜在变量无法编码全局3D几何）。我们将两者追溯到同一个根本原因：在潜在令牌级别缺乏共享的符号中间语言来对齐语言、几何和像素。我们提出了DRIVE-CHOREO，一种由LLM编排的多智能体世界模型，将可控的多视角视频生成重新定义为潜在编排。三个Qwen2.5-VL智能体——一个导演解析用户意图为结构化WorldScript，一个制图师将其嵌入空间锚定的布局令牌，一个审计员将跨视图批评反馈作为辅助监督——共同创作一个单一的位置感知令牌序列。该序列通过视图-时间置换与多视角视频进行共压缩，在3D VAE的卷积感受野内强制执行相机间几何。在nuScenes上，DRIVE-CHOREO以具有竞争力的FVD（45.7）设定了新的多视图一致性和BEV mAP（21.6）的最先进水平；一个仅在我们的合成数据上训练的检测器在真实验证集上获得了+2.4 NDS的提升，验证了下游实用性。

## Abstract
Generative world models for autonomous driving face two unresolved tensions: heterogeneous control injection, where free-form language, HD-maps, trajectories, and camera poses reside in incompatible representational spaces, and post-hoc cross-view fusion, where per-camera latents fail to encode global 3-D geometry. We trace both to a single root cause: the absence of a shared symbolic interlingua aligning language, geometry, and pixels at the latent-token level. We present DRIVE-CHOREO, an LLM-choreographed multi-agent world model that recasts controllable multi-view video generation as latent choreography. Three Qwen2.5-VL agents - a Director parsing user intent into a structured WorldScript, a Cartographer grounding it into spatially-anchored layout tokens, and an Auditor feeding cross-view critiques back as auxiliary supervision - jointly author a single position-aware token sequence. This sequence is co-compressed with the multi-view video via a view-time permutation that enforces inter-camera geometry within the convolutional receptive field of a 3-D VAE. On nuScenes, DRIVE-CHOREO sets new state-of-the-art multi-view consistency and BEV mAP (21.6) with competitive FVD (45.7); a detector trained purely on our synthetic data gains +2.4 NDS on the real validation split, validating downstream utility.

- **核心问题**：现有自动驾驶生成式世界模型面临两大对立难题：
    1. **异构控制注入**：自由形式语言、高清地图（HD-map）、轨迹和相机姿态等控制条件属于不同表示空间，现有方法须通过ControlNet分支和交叉注意力适配器分别处理，导致两个控制流在扩散模型中后验调和困难。
    2. **后处理跨视图融合**：每个相机独立经过3D VAE编码，仅在后处理阶段通过跨视图注意力融合，导致相机局部潜在变量无法编码全局3D几何，产生跨视角漂移、光度闪烁和物体瞬移。
- **根源**：缺少在潜在令牌级别对齐语言、几何和像素的共享符号中间语言。
- **整体含义**：提出OmniDrive，通过LLM编排的多智能体世界模型将可控多视图视频生成重新定义为“潜在编排”，首次在单个位置感知令牌网格中统一语言、几何和像素。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- 将可控驾驶生成视为多智能体潜在编排问题，三个Qwen2.5-VL智能体（建筑师、制图师、审计员）协作生成一个位置感知令牌序列，该序列与多视角视频通过视图-时间置换进行共压缩，使得条件与视觉令牌在相同的坐标网格中。

### 关键技术细节

1. **多智能体WorldScript编排**：
   - **建筑师（Director）**：将用户自由形式提示解析为结构化JSON（WorldScript），包含全局状态、自我轨迹、代理列表、道路拓扑。通过冻结的Qwen2.5-VL文本头+线性投影生成语义令牌Csem。
   - **制图师（Cartographer）**：将WorldScript接地到几何：为每个相机-时间对渲染稀疏布局图像（HD地图栅格化、3D边界框投影、自我轨迹带），与RGB流经历相同的视图-时间置换并通过同一VAE编码，生成几何令牌Cgeo（与视觉令牌共享相同位置索引）。相机外参通过Plücker射线嵌入为单个令牌ccam。
   - **审计员（Auditor）**：训练期间，审计员消费解码的多视图画面与WorldScript，生成结构化批评R（每对相机的一致性评分和故障模式标记），嵌入为审查令牌Crev，添加辅助目标。推理时可选用于测试时修正。

2. **潜在共压缩**：
   - **视图-时间置换Π**：将(h, t)映射为伪时间˜t=(h-1)T+t，将RGB立方体和几何立方体排列后输入单个3D VAE。所有权重迁移，仅微调最后残差块。
   - **共享噪声**：同一物理时刻的所有相机共享噪声端点z1，降低跨视图光度方差。
   - **视图块感知填充（VBP）**：对3D卷积核应用二进制掩码，防止伪时间卷积跨越视图边界（当核宽度rt≥N时），避免特征泄漏。
   - **理论保证**：Π保持Lipschitz常数不变；VBP减少相机间方差（约60%）；等变平滑项Lsm仅惩罚非仿射运动。

3. **编排生成在MM-DiT中**：
   - 共压缩潜在z0被3D卷积patch化为令牌X，所有条件串联成单一确定性序列S=[X; Csem; Cgeo; Ctmp; Crev]，使用3D RoPE位置索引。
   - 在MM-DiT中，增加几何注意力偏置logγgeo Mgeo，γgeo=4为经验最优。
   - 使用修正流ODE，K=2步Heun积分。

4. **训练**：
   - 损失L = LCFM + λsm Lsm + λrev Lrev，其中Lrev使速度差异与审计员评分对齐。
