## 摘要
4D时空推理，联合建模3D空间结构和时间演化，对于理解动态世界和实现具身交互至关重要。尽管当前的多模态大语言模型（MLLMs）在静态场景理解和粗粒度4D任务中表现出强大的能力，但在连续动态场景感知方面仍有显著局限性，尤其是在跟踪动态对象证据以实现连贯的4D时空推理方面。这一缺陷主要源于依赖稀疏的帧级观察，将连续动态线索碎片化，使模型无法区分真实对象动态与相机引起的表观运动。受人类在补偿视角变化的同时跟踪动态线索的启发，我们提出了DynTrace，一个无需训练的4D时空推理框架，包含两个互补组件。动态轨迹可视化（DTV）将世界坐标轨迹重新投影到图像平面，提供基于几何信息的视觉先验，以区分真实对象动态与相机引起的表观运动。同时，动态跟踪标记（DT-Token）组织成动态跟踪图（DTG），跟踪对象级动态线索、轨迹演变和关键时刻，为连贯的4D推理维护连续的动态对象证据。这两个组件共同为MLLMs提供持续跟踪的动态对象证据，基于几何信息的视觉先验和结构化的时空轨迹。DynTrace持续改进开源MLLMs，在Dyn-Bench、VLM4D和DSI-Bench上取得了最先进的结果，验证了跟踪动态对象证据对于稳健的4D时空推理的重要性。

## Abstract
4D spatio-temporal reasoning, jointly modeling 3D spatial structure and temporal evolution, is essential for understanding dynamic worlds and enabling embodied interaction. While current Multimodal Large Language Models (MLLMs) show strong capabilities in static scene understanding and coarse-grained 4D tasks, they still have notable limitations in continuous dynamic scene perception, especially in tracking dynamic object evidence for coherent 4D spatio-temporal reasoning. This shortcoming stems mainly from relying on sparse frame-level observations, fragmenting continuous dynamic cues and leaving models unable to disentangle genuine object dynamics from camera-induced apparent motion. Inspired by humans tracking dynamic cues while compensating for viewpoint changes, we propose DynTrace, a training-free framework for 4D spatio-temporal reasoning with two complementary components. Dynamic Trajectory Visualization (DTV) reprojects world-coordinate trajectories onto the image plane, providing geometry-informed visual priors that disentangle genuine object dynamics from camera-induced apparent motion. Meanwhile, the Dynamic Trace Token (DT-Token), organized into a Dynamic Trace Graph (DTG), tracks object-level dynamic cues, trace evolution, and key moments, maintaining continuous dynamic object evidence for coherent 4D reasoning. Together, these two components equip MLLMs with continuously tracked dynamic object evidence, grounded in geometry-informed visual priors and structured spatio-temporal traces. DynTrace consistently improves open-source MLLMs, achieving state-of-the-art results on Dyn-Bench, VLM4D, and DSI-Bench, validating the importance of tracking dynamic object evidence for robust 4D spatio-temporal reasoning.

  - **动态源混淆**：模型无法区分物体自身的真实运动与相机自身运动（如平移、旋转、变焦）引起的表观运动，导致运动来源判断错误。
  - **动态轨迹丢失**：模型依赖稀疏帧级采样（如均匀抽16帧），导致物体连续运动轨迹被碎片化，注意力漂移，动态证据链断裂。
- **动机**：人类4D认知能够同时补偿自我运动并持续跟踪感兴趣实体，这启发作者设计一种无需训练的框架，将连续动态对象证据显式注入MLLM推理过程。
- **意义**：解决该瓶颈对于具身智能、机器人交互、开放世界理解等需要持续时空感知的领域至关重要。

## 2. 方法论：DynTrace框架（核心思想、关键技术细节、流程）

### 核心思想
将原始视频转换为显式的动态对象证据（几何信息视觉先验 + 结构化文本轨迹），再送入MLLM推理，避免模型从稀疏帧中自行重建长期动态。

### 整体流程（三阶段）

**阶段1：动态对象提取**
- ① 使用 Qwen3-VL-8B 分解用户查询，提取与查询相关的实体描述 E = {e1, e2, ..., eN}。
- ② 利用 WAFT 估计稠密光流，通过仿射变换拟合背景流，得到残差运动场（实现 **相机运动消除**）。
- ③ 对残差运动点进行聚类，生成动态对象候选框，并用点提示 + 文本提示驱动 SAM3 进行时序一致的分割与跟踪，输出每个对象每帧的动态掩膜 M。

**阶段2：时空动态编码**
- 使用 DA3 (Depth Anything 3) 估计每帧深度和相机位姿。
- 对每个对象 i，利用掩膜质心、深度、相机内参和相机-世界变换，恢复世界坐标轨迹：  
  p_{t}^{i} = T_{t}^{c→w} [ z_{t}^{i} K_{t}^{-1} (u_{t}^{i}, v_{t}^{i}, 1)^{T} ]
- 从轨迹和相机位姿中提取三种基础信息：**对象轨迹**、**相机行为**（稳定/平移/变焦）、**关系演化**（对象对间距离与方位随时间变化）。

**两个互补组件**：
- **动态轨迹可视化 (DTV)**：视觉分支。将世界坐标轨迹的局部运动方向重新投影到当前图像平面，在对象掩膜上绘制方向箭头。箭头方向源于真实世界运动，而非原始图像位移，因此能有效区分真实运动与相机引起的表观运动。
- **动态轨迹图 (DTG)**：文本分支。将视频划分为 K 个时间窗口，每个窗口内对每个对象提取三类语义字段 → 编码为**对象标记**；对每对对象提取关系演化 → 编码为**关系标记**。所有标记与相机摘要组织成窗口级图结构，并序列化为紧凑文本。

**阶段3：表示整合与推理**
- 将用户查询、DTV 图像序列、DTG 文本序列同时输入目标 MLLM：  
  ŷ = MLLM(q, D, G)
- MLLM 直接利用 DTV 解决视角敏感问题（如“物体是向哪个方向运动？”），利用 DTG 的轨迹和关键事件解决长期关系推理（如“两个物体是靠近还是远离？”）。

### 关键技术细节
- **相机运动消除**：取光流，拟合背景仿射变换，计算残差流，保留仅由物体自身运动引起的运动区域。
- **轨迹重投影**：从世界轨迹取短探针点 (p_{t}^{i} + λ·单位运动方向)，用相机投影映射到像素平面，生成视觉箭头。
- **DT-Token 结构**：每个对象标记包含动态线索（位置、速度、航向、相机相对状态）、轨迹演化（方向、加速度趋势）、关键时刻（转弯、急停、最接近等）。关系标记类似，但针对对象对的距离、方位及交互事件。

## 6. 主要结论与发现

1. **显式跟踪动态对象证据至关重要**：DynTrace在三个基准上一致提升所有backbone（+3.0%~+6.1%），且8B+DynTrace超过32B基线，表明良好的动态证据比更大模型更有效。
2. **DTV与DTG互补**：DTV主要解决动态源混淆（在相机-物体类别提升明显），DTG主要解决动态轨迹丢失（在物体间关系类别提升明显），两者结合效果最优（+5.0% vs 单独+2.2%/+0.9%）。
3. **DT-Token的三个组件缺一不可**：动态线索提供主体增益，轨迹演化改善连续性，关键时刻记录决定性事件，三者协同达到最佳。
4. **中等帧数（16帧）最优**：帧数太少损失证据，帧数太多冗余，DTG已提供紧凑轨迹，16帧为效率与效果平衡点。
5. **对遮挡和中间感知误差鲁棒**：即使有临时遮挡或预测不完美的掩膜/深度，仍能提升。

## 7. 优点

- **无需训练**：直接套用现有预训练模型，即插即用，便于推广应用。
- **显式建模运动解耦**：通过世界坐标轨迹重投影，从根本解决了相机运动引起的表观运动误导。
- **结构化文本轨迹**：DTG以图方式组织对象/关系动态，既保持长期连续性，又压缩为LLM友好的紧凑文本，避免长序列处理问题。
- **双模态互补**：视觉箭头（DTV）提供直观几何先验，文本轨迹（DTG）提供精确数值与事件，两者协同适用于不同问题类型。
- **全面实验验证**：在多个基准和多种backbone上验证，且设计了失败模式子集分析，直接对应该文识别的问题瓶颈。
- **可解释性**：DTV箭头和DTG文本使得最终推理过程可追溯、可验证。

## 8. 不足与局限

- **中间依赖**：依赖多个外部模型（WAFT、SAM3、DA3、Qwen3-VL）的输出质量，若这些模型失效（如严重遮挡、极端光照、快速运动模糊），证据质量下降。虽有GT vs pipeline对比显示鲁棒，但未在极端场景下系统测试。
- **组件计算开销未量化**：论文未提供推理时间或FLOPs比较，实际部署时多阶段串联可能增加延迟。
- **仅针对查询相关实体**：当前仅提取查询中提及的实体，对于隐式相关或需要全面场景理解的推理可能遗漏信息。
- **部分级动作的局限性**：定性案例显示，当问题依赖于身体部位（如“手相对于狐狸的位置”）时，DynTrace将整个对象作为整体建模，缺乏部位级运动建模。
- **自我中心视角转换**：当前证据围绕对象轨迹和相机相对上下文，对于需要转换到物体第一人称视角的问题，证据不够充分。
- **实验 backbone 多样性有限**：主要测试了Qwen3-VL系列和InternVL3.5系列，未评估LLaVA、VideoLLaMA等更大模型（但文中对比了这些模型作为基线）。
- **可改进方向**：扩展为对象中心视角变换、细粒度部位运动建模、多查询联合推理等。
