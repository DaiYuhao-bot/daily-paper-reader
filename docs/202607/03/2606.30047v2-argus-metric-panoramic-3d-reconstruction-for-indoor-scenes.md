## 摘要
由于缺乏大规模全景RGB-D训练数据，面向全景数据的度量前馈三维重建仍未被充分探索。我们提出了Realsee3D，一个包含10K室内场景（1K真实、9K合成）的混合数据集，拥有299K个全景视点和精确的度量标注，以及Argus——一个在该数据集上训练的用于度量全景三维重建的前馈网络。在Realsee3D稀疏无序采集设置中，不恰当的坐标锚点可能导致全局位姿漂移。Argus通过一个学习到的共视模块来解决此问题，该模块选择几何最优参考视图来锚定度量世界坐标系。为了进一步改进多任务学习，我们将双向像素到世界映射分解为可解释的子步骤，并配以逐步骤监督和跨坐标联合约束，从而增强预测分支之间的几何一致性。在Realsee3D基准测试中，Argus在相机位姿估计、深度估计和点云重建方面达到了最先进的度量性能。项目页面：https://argus-paper.realsee.ai。

## Abstract
Metric feed-forward 3D reconstruction for panoramic data remains under-explored due to the lack of large-scale panoramic RGB-D training data. We present Realsee3D, a hybrid dataset of 10K indoor scenes (1K real, 9K synthetic) with 299K panoramic viewpoints and precise metric annotations, and Argus, a feed-forward network trained on it for metric panoramic 3D reconstruction. In the sparse unordered capture setting of Realsee3D, a poorly chosen coordinate anchor can cause global pose drift. Argus addresses this with a learned covisibility module that selects the geometrically optimal reference view to anchor the metric world frame. To further improve multi-task learning, we decompose the bidirectional pixel-to-world mapping into interpretable sub-steps with per-step supervision and cross-coordinate joint constraints, reinforcing geometric consistency across prediction branches. On the Realsee3D benchmark, Argus achieves state-of-the-art metric performance in camera pose estimation, depth estimation, and point cloud reconstruction. Project page: https://argus-paper.realsee.ai.

# 论文核心问题与整体含义（研究动机和背景）

全景图像因其360°视场角而在室内三维采集中被广泛采用，但此前缺乏大规模、高质量、带精确度量标注的全景RGB-D训练数据，导致现有前馈三维重建方法（如VGGT、DUSt3R等）主要面向透视图像，直接迁移到全景数据时性能严重下降。此外，在稀疏、无序的全景采集场景中，如果锚定坐标系的参考视图选择不当（例如位于场景角落或边界），会引起全局位姿漂移和重建不一致。为此，论文：

- 构建了大规模混合数据集 **Realsee3D**（10K室内场景，299K全景视点），包含1K真实扫描和9K程序化合成场景，提供精确的RGB-D、位姿和度量注释。
- 提出前馈网络 **Argus**，通过学习共视（covisibility）模块自动选择几何最优的参考视图锚定世界坐标系，并将全景模型下的双向像素-世界映射分解为多个可解释子步骤，分别施加监督并加入跨坐标联合约束，从而增强多任务学习的几何一致性。

# 方法论：核心思想与关键技术细节

## 核心思想

1. **学习式共视参考视图选择**：利用一个轻量级的Covisibility Transformer预测各视图的共视得分，选出全局连通性最强的视图作为参考帧（替代固定的启发式规则），抑制位姿漂移。
2. **全景几何分解监督（Overcomplete Geometric Factorization）**：将像素到世界的变换分解为一系列中间表示（单位球面点、相机坐标系点、旋转后点、世界坐标系点），每个中间表示都由独立的DPT头预测，并逐步骤监督；同时加入跨坐标的几何联合损失，促进分支间一致性。
3. **度量预测**：由于ERP投影模型固定（无内参耦合），且数据集提供一致的度量尺度（LiDAR和PBR渲染），直接预测度量深度，无需额外的尺度估计模块。

## 关键技术细节

- **网络架构**：基于DINOv2提取patch token；添加共视token后经2层交替注意力共视Transformer得到各视图共视得分，选最高分视图作为参考；然后序列经24层交替注意力几何Transformer；最后通过MLP回归位姿（9维：平移、四元数、旋转/平移置信度），DPT头预测深度、多坐标系点图和像素级置信度。
- **几何变换公式**：给定ERP图像像素坐标(u,v)映射到球面（经度ϕ、纬度θ），单位球面点Pu；摄像机点Pc = D ⊙ Pu；旋转点Pr = R * Pc；世界点Pw = Pr + t；反向同理。深度由∥Pc∥计算。
- **损失函数**：包括共视损失（BCE）、相机损失（旋转+平移+置信度）、深度损失（aleatoric uncertainty + 梯度项）、多坐标系点图损失（点图+法线+置信度）、几何联合损失（对相邻变换步骤的一致性监督）。总损失加权求和。

# 主要结论与发现

1. **Argus在Realsee3D基准上全面领先**：在相机位姿估计中，ATE在真实子集上降低28%以上，在合成子集上降低69%以上（对比MapAnything360）；深度估计在所有指标上最优（如合成子集AbsRel 0.019 vs 基线的0.025）；点云重建在度量评估下优势明显。
2. **参考视图学习有效抑制位姿漂移**：去掉后ATE从0.027升至0.060，翻倍以上。
3. **几何分解监督增强多任务协同**：逐步去除各点图损失导致深度和位姿性能持续下降，特别是相机坐标点图监督作用最大。
4. **泛化能力**：能处理AI生成的艺术风格、弱共视、多楼层、远距及小尺度室外场景，但强于透视基线的零样本泛化。

# 优点

- **数据集贡献**：Realsee3D是迄今最大规模的全景RGB-D室内数据集，混合真实与合成数据，覆盖10K场景，质量高、标注精确。
- **方法创新**：学习式共视参考视图选择避免了手工启发式的局限性，并保持近似排列等价性；全景几何分解监督将复杂变换拆解为可监督的中间步骤，降低了优化难度，提升了多任务协同。
- **实验全面**：包含多种任务、多种对齐方式、消融和效率分析，对比方法公平。
- **实用性好**：多DPT头在推理时可关闭，无额外开销；支持从2到128+帧输入，显存增长可控。

# 不足与局限

- **数据域局限**：训练数据集中于室内结构化环境，零样本泛化到无界室外、航拍等场景能力不足（论文已承认）。
- **规模扩展性**：当输入全景图像数量达到数百到上千时，显存和计算时间增长，尚未支持非常大规模场景的端到端重建。
- **实验覆盖**：未与传统SfM/MVS方法进行定量对比（由于它们在Realsee3D上运行缓慢且结果不可用），但论文已说明原因。
- **应用限制**：依赖ERP投影，对非标准摄像机的适配未讨论；深度/点图监督需要精确的逐像素真值，限制了无监督/自监督场景的应用。
