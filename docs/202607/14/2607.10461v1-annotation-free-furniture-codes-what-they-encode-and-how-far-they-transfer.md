## 摘要
基于布局的3D场景合成器使用两个人工标注通道来放置每个对象：一个类别标签和一个规范姿态约定。我们探究一个源自对象几何的单自监督标记能否替代两者，并将此类标记直接作为一种表示进行研究，与任何合成器解耦。一个有限标量量化（FSQ）点云自编码器在放置的3D-FUTURE家具上进行倒角训练，不使用任何标签或姿态标注。仅从编码中，诊断探针恢复了细粒度类别（62.6±0.5%）、超类别（85.6±1.3%）和偏航角（52.7±0.5度）。将倒角目标从旋转点云替换为未旋转点云会消除偏航信号，同时提高类别恢复率，表明编码的旋转内容可由训练目标设定。跨资产库的缩放需要可迁移的编码；在未见过的数据集（ShapeNet）上，对齐程度取决于类别：盒状家具能够迁移，而有机形状家具则不能，一种无视目标的增广部分弥补了这一差距。

## Abstract
Layout-based 3D scene synthesizers place each object using two human-annotated channels: a categorical class label and a canonical-pose convention. We ask whether a single self-supervised token derived from object geometry can replace both, and study such tokens directly as a representation, decoupled from any synthesizer. A Finite Scalar Quantization (FSQ) point-cloud autoencoder is chamfer-trained on placed 3D-FUTURE furniture with no labels or pose annotations. Diagnostic probes recover fine-category (62.6 +/- 0.5%), super-category (85.6 +/- 1.3%), and yaw (52.7 +/- 0.5 deg) from the codes alone. Swapping the chamfer target from the rotated to the un-rotated point cloud collapses the yaw signal while raising class recovery, showing the codes' rotation content can be set by the training objective. Scaling across asset libraries needs codes that transfer; on an unseen dataset (ShapeNet), alignment is category-dependent: box-like furniture transfers, organically-shaped furniture does not, and a target-blind augmentation partly closes the gap.

### 6. 论文的主要结论与发现

1. **无标签编码包含类别和偏航信息**：从单码恢复细类62.6%、超类85.6%、偏航角平均误差52.7°（中位数10.2°），显著高于随机/多数基线。
2. **旋转编码由训练目标控制**：将Chamfer目标从旋转点云改为规范（未旋转）点云后，偏航探针坍塌至模态基线（79.4°），类别恢复提升（超类85.6%→89.8%）。表明旋转内容是数据诱导的，可通过一个布尔开关设计。
3. **偏航双峰分布源于前后对称**：约20%样本出现180°翻转，由对称物体（如桌子、柜子）的码对旋转不变性导致。
4. **跨数据集迁移依赖于类别几何结构**：盒状类别（橱柜、书架、桌子）对齐良好（r≈0.9-1.6），有机形状（床、沙发、椅子）偏离3.2-5.1倍类内间距。检索P@1随距离比r平滑下降，在r≈3处接近随机。
5. **通用增广可部分缩小差距**：加入噪声/丢弃/体素快照后，盒状类别稳定转移，有机形状类别的距离比向r=3边界靠拢，但种子间差异大（沙发2.3/4.0，床3.4/2.4，椅子4.3/2.9），无法确保一致跨越阈值。
6. **量化代价合理**：从连续潜变量到离散500码，超类精度损失6.5pp，偏航角损失7.3°。FSQ相比VQ-VAE利用率更高（96.3% vs 7.5%），但结论谨慎（VQ-VAE可调参改善）。

### 7. 优点：方法或实验设计上的亮点

- **问题定义清晰**：将自监督几何码的特征刻画与下游合成器解耦，定位为“代码内容是什么”这一先决问题，避免混合评估。
- **方法简洁有效**：使用FSQ（固定格量化，无学习码本）替代复杂VQ-VAE，训练快、利用率高，易于复现。
- **精心设计的控制实验**：仅改变Chamfer目标（旋转 vs 规范），保持架构/优化器不变，直接检验旋转编码的来源，结论有力。
- **量化转移程度**：提出距离比r作为连续度量，结合检索P@1，清晰展示了转移的连续性与类别依赖性，优于简单二值分类。
- **探针协议高效**：可在约2分钟内完成对90k编码的类别和偏航评估，可作为tokenizer质量的单元测试。
- **领域增广的合理性**：使用与目标数据集无关的通用数字化增广（噪声、丢弃、体素快照），不泄露目标数据信息，保持真正的“未见数据集”测试。

### 8. 不足与局限

- **无下游场景合成评估**：只分析编码内容，未训练标签自由的场景生成器，因此实际端到端性能未知（如FID、布局质量等）。
- **姿态与形状纠缠**：旋转目标编码中偏航与形状混合在一个约9位标记中，无法独立控制姿态（改变物体朝向需跳转到可能改变形状的另一个编码）。规范目标方案消除了纠缠但不再无标注。
- **仅考虑偏航旋转**：3D-FRONT只使用绕垂直轴（yaw）旋转，结论不适用于完整SO(3)旋转。
- **单一数据集对**：迁移分析仅限3D-FUTURE ↔ ShapeNet，一个自编码器；可能不足以泛化到更多资产库（如Objaverse、3D Warehouse）。
- **种子覆盖不均**：核心Table 1有3次重复，但跨数据集、增广、VQ-VAE、编码器比较多为单种子，统计力量有限。
- **训练偏航分布混淆**：类条件偏航探针中，可恢复的类（沙发、椅子）在训练中可能看到更多多样化偏航，与几何对称性混淆。匹配偏航控制只在评估时调整，未在训练时均衡。
- **训练分布混淆未完全分离**：几何可观测性与训练偏航多样性的区分需要重训偏航均衡数据（未做）。
- **无外部基准对比**：未将单9位编码与高层次自监督点云编码器（如Point-BERT、Point-MAE）公平比较（它们使用多token、更高容量）；仅与连续潜变量对比量化损失。
- **增广效果种子依赖**：有机形状类别的转移改善种子可变，无法稳健跨越r<3阈值，表明仍需更多训练数据或复杂数据策略。
